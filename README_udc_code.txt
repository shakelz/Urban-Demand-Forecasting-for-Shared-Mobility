
Cross-Modal Shared Mobility Demand Forecasting  Code

Project Title
Cross-Modal Shared Mobility Demand Forecasting

Overview
This project implements a cross-modal forecasting framework for short-term shared mobility demand prediction. The main focus is bike-sharing demand forecasting using historical demand, temporal features, weather variables, calendar indicators, holiday/weekend effects, demand regimes, and proxy neighbourhood-context features.

The code supports the experimental evaluation of the thesis research questions, including model comparison, multi-horizon forecasting, contextual modality contribution analysis, spatial heterogeneity analysis, abrupt demand-shift testing, transferability evaluation, explainability analysis, and operational decision-support simulation.

Main Objective
The objective of the code is to evaluate whether combining multiple contextual modalities can improve short-term shared mobility demand forecasting compared with traditional and temporal-only forecasting baselines.

Research Questions Covered
RQ1: Cross-modal fusion forecasting performance
RQ2: Contextual modality contributions by demand regime
RQ3: Neighbourhood embeddings and spatial heterogeneity
RQ4: Forecasting abrupt urban demand shifts
RQ5: Transferability across urban mobility contexts
RQ6: Explainable contextual forecast interpretation
RQ7: Operational decision support through forecasts

Dataset
The project uses hourly bike-sharing demand data. The dataset contains demand records and related contextual variables such as time, weather, working day, holiday/weekend, and seasonal information.

If the dataset is not already provided in the /data folder, the notebook may automatically download or load the dataset from a public source. Make sure that the dataset file path in the notebook matches the dataset location in your environment.

Input Features
The forecasting framework uses the following feature groups:

1. Historical demand features
   - Previous demand values
   - Lagged demand
   - Rolling demand statistics

2. Temporal features
   - Hour of day
   - Day of week
   - Month
   - Season
   - Cyclical time features

3. Weather features
   - Temperature
   - Humidity
   - Wind speed
   - Weather condition

4. Calendar features
   - Working day
   - Weekend
   - Holiday

5. Proxy neighbourhood-context features
   - CBD
   - Residential
   - Tourist
   - Transit hub
   - University
   - Mixed-use

Models Compared
The code compares the proposed Full Cross-Modal Fusion model against several baseline and partial contextual models:

1. Historical Average
2. Temporal Only
3. Temporal + Weather
4. Temporal + Calendar
5. Temporal + Neighbourhood
6. Weather + Calendar
7. Full Cross-Modal Fusion

Evaluation Metrics
The forecasting models are evaluated using:

RMSE: Root Mean Square Error
MAE: Mean Absolute Error
WAPE: Weighted Absolute Percentage Error
Peak-MAE: Mean Absolute Error during peak-demand periods
RMSE Gain: Percentage improvement over baseline models

How to Run the Code
1. Open the required notebook in Kaggle Notebook.
2. Check that the dataset path is correct.
3. Run the cells from top to bottom.
4. The notebook will preprocess the data, train models, evaluate results, and save figures/tables.
5. Repeat the same process for each RQ notebook if the code is separated by research question.

Recommended Running Order
Run the notebooks in the following order:

1. RQ1 notebook
2. RQ2 notebook
3. RQ3 notebook
4. RQ4 notebook
5. RQ5 notebook
6. RQ6 notebook
7. RQ7 notebook

Reproducibility Notes
To reproduce the results:
- Use the same dataset.
- Keep the same train-test split.
- Run all preprocessing steps in the same order.
- Use the same forecasting horizons.
- Use the same model configurations.
- Use the same evaluation metrics.
- Do not shuffle the time-series data before splitting.
